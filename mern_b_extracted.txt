MERN Deployment 
 
These instructions are for Linux, specifically Ubuntu 22 was used. 
 
First, you need a Digital Ocean account. You should already have this, but you can either remove mysql, php, 
and apache2 or you can create a new droplet (the latter is preferred). 
 
You will need to point your domain to the droplet IP address. 
 
Create a Linux droplet. Don’t create a LAMP droplet, just create from the main Digital Ocean dashboard. 
 
I am using ssh to login as shown here. (I had to install SSH for my Windows 10 machine, Windows 11 and 
Macs have it built in.) I might use Putty from my server.

Unfortunately, you cannot login because there is no API server listening. Let’s do that next. 
 
First, we will create a directory on the server and then copy our server files into it. 
 
From /var to the following: 
mkdir cardsServer 
cd cardsServer 
 
Now the path to your cards server is /var/cardsServer (remember that Linux is case sensitive). 
 
We will not copy the node_modules directory since it is very large and we can re-install it on the server. 
 
From the cards directory: 
scp -r server.js root@cop4331-5.com:/var/cardsServer 
scp -r package.json root@cop4331-5.com:/var/cardsServer 
scp -r package-lock.json root@cop4331-5.com:/var/cardsServer 
scp -r .env root@cop4331-5.com:/var/cardsServer 
navigate to /var/cardsServer and type npm install (this rebuilds the node_modules directory) 
edit package.json and replace nodemon with node 
 
Navigate to /var/cardsServer 
Run npm install -g pm2 
Run pm2 start server.js --name "express-server" 
Run pm2 startup 
Run pm2 save 
 
The front end cannot contact the back end because the front end is actually running on your local machine and 
trying to use port 5000 on the back end server. Open port 5000 with sudo ufw allow 5000

GitHub Actions Workflow File 
Create this file in your repository at .github/workflows/deploy.yml: 
name: Deploy MERN Application 
 
on: 
  push: 
    branches: [ main ] 
 
jobs: 
  deploy: 
    runs-on: ubuntu-latest 
     
    steps: 
    # Checkout the repository code 
    - name: Checkout code 
      uses: actions/checkout@v3 
 
    # Setup Node.js 
    - name: Setup Node.js 
      uses: actions/setup-node@v3 
      with: 
        node-version: '22.18' 
        cache: 'npm' 
        cache-dependency-path: | 
          package-lock.json 

--- PAGE 7 ---

          frontend/package-lock.json 
 
    # Install and build frontend 
    - name: Install frontend dependencies 
      working-directory: ./frontend 
      run: npm ci 
 
    - name: Build frontend 
      working-directory: ./frontend 
      run: npm run build 
 
    # Deploy frontend to server 
    - name: Deploy frontend to server 
      uses: appleboy/scp-action@v0.1.5 
      with: 
        host: ${{ secrets.SERVER_HOST }} 
        username: ${{ secrets.SERVER_USER }} 
        password: ${{ secrets.SERVER_PASSWORD }} 
        port: ${{ secrets.SERVER_PORT }} 
        source: "frontend/dist/*" 
        target: "/var/www/html" 
        strip_components: 2 
        rm: true 
 
    # Deploy backend files to server 
    - name: Deploy backend files 
      uses: appleboy/scp-action@v0.1.5 
      with: 
        host: ${{ secrets.SERVER_HOST }} 
        username: ${{ secrets.SERVER_USER }} 
        password: ${{ secrets.SERVER_PASSWORD }}

One-Time Server Setup Instructions 
1. Connect to your server 
bash 
ssh root@cop4331-3.com 
2. Ensure Node.js 22.18 is installed 
bash 
# Check current version 
node --version 
 
# If not 22.18, install it using NodeSource repository 
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - 
sudo apt-get install -y nodejs 
3. Install PM2 globally (if not already installed) 
bash 
npm install -g pm2 
4. Create necessary directories 
bash 
# Ensure backend directory exists 
mkdir -p /var/cardsServer 
 
# Ensure frontend directory exists with proper permissions 
mkdir -p /var/www/html 
chown -R www-data:www-data /var/www/html 
chmod -R 755 /var/www/html 
5. Setup PM2 to start on system boot 
bash 
pm2 startup systemd 
# Follow the command it outputs 
pm2 save 
6. Ensure .env file is in place 
Create or verify your .env file exists in /var/cardsServer/.env with your environment variables. Note: The .env 
file should NOT be in your GitHub repository for security reasons. 

--- PAGE 10 ---

Example .env content: 
PORT=5000 
MONGODB_URI=your_mongodb_connection_string 
# Add other environment variables as needed 
7. Verify Nginx configuration 
bash 
# Test nginx configuration 
nginx -t 
 
# If successful, reload nginx 
systemctl reload nginx 
8. Verify firewall settings 
bash 
# Check if port 5000 is open 
ufw status 
 
# If not open, add it 
ufw allow 5000/tcp 
9. Set proper permissions for deployment 
bash 
# Ensure the deployment user can write to these directories 
chmod -R 755 /var/cardsServer 
chmod -R 755 /var/www/html 
10. Initial PM2 setup (if app not already running) 
bash 
cd /var/cardsServer 
# Only if the app isn't already running 
pm2 start server.js --name "express-server" 
pm2 save
